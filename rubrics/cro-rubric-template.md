# CRO Rubric Template v1.0

Optional consumer rubric for marketing/conversion projects. Pass via `--rubric` when the project has landing pages, pricing, onboarding, or lead generation flows.

Based on: Cialdini's principles, Fogg behavior model, pricing page research, and form optimization studies.

---

## Landing / Value Proposition Page

Identified by: first page visitors see, hero section, primary CTA

### Must Pass
- Value proposition is visible above the fold before any signup/form friction
- Primary CTA is visible without scrolling and has benefit-oriented text
- Social proof present near CTA (testimonials, logos, user counts, or case studies)
- Page communicates what happens after the CTA action (set expectations)

### Should Pass
- Secondary CTA available for lower-commitment visitors (learn more, watch demo)
- Authority signals present (certifications, press mentions, partner logos)
- Scarcity or urgency used ethically if applicable (limited offer, seats remaining)

### Deal-Breakers
- No value proposition visible — page leads with signup form without context
- CTA leads to 404 or broken page

---

## Pricing Page

Identified by: pricing tiers, plan comparison, billing toggle

### Must Pass
- 3-4 tiers clearly differentiated and aligned with user segments
- Recommended/popular tier visually distinct (highlight, badge, or emphasis)
- Annual vs monthly billing clearly shown with discount percentage
- Features grouped by outcome/use-case, not flat feature list
- Each tier CTA has specific action text (not just "Buy" on all)

### Should Pass
- Price anchoring: higher tier shown to make mid-tier look like better value
- FAQ section addressing switching, cancellation, refund near pricing
- Social proof near pricing (logos, "X companies use this plan")
- Free trial or money-back guarantee reduces perceived risk

### Deal-Breakers
- Pricing is hidden or requires contacting sales for standard plans
- Clicking a plan CTA leads to error or broken checkout

---

## Signup / Onboarding Flow

Identified by: registration form, account creation, onboarding wizard

### Must Pass
- Signup form asks only essential fields (name, email, password max for initial step)
- Social login options available (Google, SSO) if applicable
- Inline validation with clear, non-technical error messages
- Success state is clear — user knows they signed up and what to do next

### Should Pass
- Progressive profiling: heavy data collection deferred to post-signup
- Post-signup onboarding checklist or guided tour with benefit-oriented steps
- Each onboarding step explains WHY that info is needed
- Progress indicator on multi-step flows

### Deal-Breakers
- Signup requires more than 5 fields in a single step without progress indicator
- No confirmation after signup — user unsure if it worked

---

## Lead / Contact Form

Identified by: contact form, demo request, whitepaper download, newsletter signup

### Must Pass
- Form length matches intent level (newsletter: 1-2 fields, demo: 3-5 fields)
- Labels above inputs, not placeholder-only
- Privacy/consent statement present and non-intimidating
- CTA is benefit-oriented ("Get the whitepaper", "Book a call"), not "Submit"

### Should Pass
- Low-commitment alternative available (chat, FAQ, download instead of call)
- Smart defaults or pre-fill where possible (country from IP, company from email domain)
- Multi-step split for forms > 5 fields with progress indicator

### Deal-Breakers
- Form submits with no confirmation or acknowledgment
- Required fields not marked — user discovers requirements only on error

---

## Persuasion & Behavioral Design (cross-page)

Identified by: applies to all pages with conversion goals

### Must Pass
- Fogg trigger present: motivation element + ability element + prompt converge at the same point (e.g., benefit text + simple form + clear CTA together)
- Social proof within viewport of primary CTA (not buried in footer)
- No dark patterns: urgency/scarcity claims are truthful, consent is clear

### Should Pass
- Reciprocity: free value offered before asking for commitment (preview, free tool, sample)
- Consistency: small commitments before big ones (quiz → result → signup)
- Liking: brand personality and tone consistent across pages
- Unity: language creates shared identity with visitor ("Join 10K marketers like you")

### Deal-Breakers
- Deceptive urgency (fake countdown timers, fabricated scarcity)
- Consent obtained through confusion or hidden checkboxes
